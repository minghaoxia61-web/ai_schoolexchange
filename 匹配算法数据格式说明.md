## 1️⃣ 输入数据格式（posts 表）

算法函数接收一个 `posts` 数组，每个 post 对象包含以下字段：

```javascript
{
  "id": Number,                    // 自增主键
  "postId": String,                // 物品ID，格式：post_{timestamp}_{random}
  "userId": String,                // 发布者ID，格式：U{timestamp}
  "type": String,                  // 物品类型：'item'（物品）或 'skill'（技能）
  "title": String,                 // 物品标题
  "description": String,           // 物品描述
  "tradeMethod": String,           // 交易方式：'money'（金钱）、'exchange'（交换）、'both'（两者皆可）
  "price": String,                 // 价格（当 tradeMethod 为 'money' 或 'both' 时）
  "exchangeItems": String,         // 想要交换的物品（当 tradeMethod 为 'exchange' 或 'both' 时）
  "images": Array<String>,         // 图片URL数组
  "contact": String,               // 联系方式
  "status": String,                // 状态：'active'（活跃）、'completed'（已完成）、'cancelled'（已取消）
  "createdAt": String,             // 创建时间，ISO 8601 格式
  "updatedAt": String              // 更新时间，ISO 8601 格式
}
```

## 2️⃣ 输出数据格式（算法返回）

算法函数需要返回一个 `cycles` 数组，每个 cycle 对象包含以下字段：

```javascript
{
  "type": String,                  // 匹配类型：'2-cycle'（双人）、'3-cycle'（三人）、'4-cycle'（四人）
  "chain": Array<String>,          // 用户ID链，表示匹配顺序
  "weights": Array<Number>,        // 边权重数组，表示每条边的匹配强度（0-1之间）
  "score": Number                  // 总体匹配成功率（0-1之间）
}
```

### 输出数据示例：

```javascript
const cycles = [
  {
    "type": "2-cycle",
    "chain": ["U1704067100000", "U1704067200000"],
    "weights": [0.85, 0.80],
    "score": 0.825
  },
  {
    "type": "3-cycle",
    "chain": ["U1704067100000", "U1704067200000", "U1704067300000"],
    "weights": [0.75, 0.80, 0.70],
    "score": 0.75
  }
]
```

## 4️⃣ 匹配逻辑说明

### 匹配类型：
- **2-cycle（双人交换）**：A → B → A
  - A 有 B 想要的物品
  - B 有 A 想要的物品
  
- **3-cycle（三人循环交换）**：A → B → C → A
  - A 有 B 想要的物品
  - B 有 C 想要的物品
  - C 有 A 想要的物品
  
- **4-cycle（四人循环交换）**：A → B → C → D → A
  - 类似三人循环，但涉及四个人

### 权重计算建议：
- **边权重（weights）**：表示两个用户之间匹配的强度
  - 可以基于：物品相似度、描述匹配度、用户偏好等
  - 取值范围：0-1
  
- **总体匹配成功率（score）**：
  - 可以是边权重的平均值、最小值或其他组合方式
  - 取值范围：0-1

## 6️⃣ 注意事项

1. **只处理 active 状态的物品**：算法接收到的 posts 已经过滤了非 active 的物品
2. **支持多种交易方式**：需要考虑 'money'、'exchange'、'both' 三种交易方式
3. **支持物品和技能**：type 字段区分物品和技能
4. **返回所有可能的匹配**：算法应该返回所有可能的匹配环，由前端展示给用户选择
5. **性能考虑**：建议优先返回高质量匹配（score 较高的）

## 7️⃣ 数据库表结构

### 用户表（users）
```sql
CREATE TABLE users (
  id INT PRIMARY KEY AUTO_INCREMENT,
  userId VARCHAR(50) UNIQUE NOT NULL,
  userName VARCHAR(100),
  phone VARCHAR(20),
  avatar VARCHAR(255),
  createdAt TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updatedAt TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);
```

### 物品发布表（posts）
```sql
CREATE TABLE posts (
  id INT PRIMARY KEY AUTO_INCREMENT,
  postId VARCHAR(50) UNIQUE NOT NULL,
  userId VARCHAR(50) NOT NULL,
  type ENUM('item', 'skill') NOT NULL,
  title VARCHAR(200) NOT NULL,
  description TEXT,
  tradeMethod ENUM('money', 'exchange', 'both') NOT NULL,
  price VARCHAR(50),
  exchangeItems VARCHAR(500),
  images JSON,
  contact VARCHAR(100),
  status ENUM('active', 'completed', 'cancelled') DEFAULT 'active',
  createdAt TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updatedAt TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  FOREIGN KEY (userId) REFERENCES users(userId),
  INDEX idx_userId (userId),
  INDEX idx_createdAt (createdAt)
);
```

### 匹配结果表（matches）
```sql
CREATE TABLE matches (
  id INT PRIMARY KEY AUTO_INCREMENT,
  matchId VARCHAR(50) UNIQUE NOT NULL,
  userId VARCHAR(50) NOT NULL,
  matchType ENUM('2-cycle', '3-cycle', '4-cycle') NOT NULL,
  chain JSON NOT NULL,
  chainNames JSON NOT NULL,
  edgeWeights JSON NOT NULL,
  successProbability FLOAT,
  status ENUM('pending', 'accepted', 'rejected', 'completed') DEFAULT 'pending',
  createdAt TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updatedAt TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  FOREIGN KEY (userId) REFERENCES users(userId),
  INDEX idx_userId (userId),
  INDEX idx_createdAt (createdAt)
);
```
